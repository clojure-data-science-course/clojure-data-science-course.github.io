Software

Here are some suggestions for the software tools that you may use in the course.

If you have any difficulty with the installation or usage – then let us talk. It is better to solve this kind of problems at early stages.

The minimum

You will at least need Java (JDK version 8 or above) and Leiningen. You may use the guide of Practical.li for installing them.

Gorilla

During the first talks, we will use Gorilla REPL, a notebook-like example originally written by Jony Hudson. It has its own visualization library and very interesting ideas regarding the creation of composable visual reports. It is rather common among Clojure data scientists.

We will also use it to share our work online.

You may use Gorilla through lein-gorilla, which requires Leiningen. Another option is gorilla-repl-docker, which requires Docker.

See some more details under the gorilla-repl stream at Zulip.

Editors

At about the 4th meeting, when we start writing larger pieces of code, it will become important to use a proper code editor.

Practical.li has nice installation guides for several editors.

At our talks we will use Atom Editor.

Other options

If you like the wonderful Emacs, then you can also use the installation guide at Higginbotham's book.

If you like Spacemacs, then you can also use the more detailed guide by Practical.li.

Another option is Nightcode - a minimalistic editor, dedicated to Clojure, which is rather easy to install.

Trying Clojure online.

For your practice, you may also use Clojure online.

  • repl.it - here you can try the language online.
